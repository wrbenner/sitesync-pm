Strategic Design Plan

The Drawings Page,
Reimagined.

A comprehensive redesign plan based on deep research into what general contractors actually need, competitive analysis of Procore, Bluebeam, PlanGrid, and Fieldwire, and the design principles of history's greatest product minds.

"Design is not just what it looks like and feels like. Design is how it works." Steve Jobs
01 — Research

What GCs Actually Need

Ranked by impact on daily work, from superintendent interviews and competitive analysis

1

Instant Access to the Right Sheet

The #1 pain point. A super on site has dirty hands, bad connectivity, and 30 seconds of patience. Finding the correct, current drawing must be effortless.

2

Crystal-Clear Version Control

"Is this Rev C or Rev E?" kills trust. Every revision must be visually unmistakable. The current set must be obvious at a glance. No ambiguity.

3

Buttery-Smooth Plan Viewing

Pinch-zoom, pan, and navigation must feel like Apple Maps — not a clunky PDF viewer. Construction drawings are large, detailed documents that demand a world-class viewer.

4

Markup That Doesn't Fight You

Pen, text, shapes, measurements — all must be one-tap away. Bluebeam sets the bar with professional-grade annotation. We need to match or exceed it.

5

Sheet-to-Sheet Hyperlinking

Detail callouts like "SEE A5.01" should be clickable links, not dead text. Procore and Fieldwire auto-detect these. We can do it with AI — better.

6

RFI / Punch / Photo Integration

Drawings are the spatial anchor for everything on a jobsite. Every issue, photo, and question should live on the plan — not in a separate tab.

02 — Diagnosis

What's Wrong Today

10 specific issues found after deep-diving the current 1,000+ line implementation

!

PDF Viewer Is Basic

react-pdf renders page-by-page at fixed resolution. No smooth zoom, no minimap, no semantic zoom. Feels like 2015.

!

Metric Cards Steal Prime Real Estate

4 stat cards at the top push actual drawings below the fold. A super doesn't care about "disciplines count."

!

340px Fixed Sidebar Wastes Space

25% of screen width locked to metadata when the user wants to SEE THE PLAN. Should be on-demand.

!

No Drawing Set Grouping

A 200-page plan set shows as 200 individual rows. No hierarchy, no discipline grouping, no spatial organization.

!

600+ Inline Style Objects

Every component is styled inline. Impossible to maintain a consistent design language or make sweeping visual improvements.

!

Table View Is Default

Construction people think spatially. Grid with large thumbnails should be default — the drawing IS the interface.

!

No Breadcrumb Navigation

Once inside a drawing, there's no spatial sense of where you are in the set. Can't quickly jump between related sheets.

!

No Keyboard Shortcuts

Only Cmd+K for search. Power users need arrow keys, space to preview, keyboard-driven markup tools.

04 — Design DNA

Lessons from the Masters

What we learned studying the best minimalist interfaces — and what we're stealing

SumatraPDF

Under 10MB. No ribbons, no sidebars, no popups. Everything subordinated to content. Lesson: eliminate every pixel not serving the user's current task.

Apple Preview

Zero onboarding needed. Annotation tools hidden until you need them. Content fills the window. Lesson: toolbars should be invisible until summoned.

PDF Expert (Readdle)

Architects call it out for speed, smoothness, and Apple Pencil fluidity. UI feels balanced and premium. Lesson: markup must feel like writing on paper, not fighting software.

Linear

Reduced 98 theme variables to 3. Monochrome with few bold accents. Not every element carries equal weight. Lesson: discipline with color, obsession with alignment.

PDFgear

Three themes, five background colors, zero clutter. Fulfills exactly what's needed. Lesson: don't add features, add calm.

"Linear Design" Trend

Clean, clutter-free, high-contrast. Single directional eye movement. Bold typography for clarity. Lesson: sequential beats decorative every time.

The Minimalist Manifesto

8 non-negotiable rules derived from the research above

1

Color Budget: Max 3 at a Time

Black/white/gray base + orange accent + one status color. Kill the rainbow of discipline colors. Use subtle tinted backgrounds instead.

2

Button Audit: 80/80 Rule

If a button isn't used by 80%+ of users on 80%+ of visits, it's hidden behind overflow. Period.

3

Zero Chrome in Viewer

Toolbar auto-hides after 3s of inactivity (like video players). Tap/move to reveal. The DRAWING is the UI.

4

3 Font Sizes Only

13px body, 16px emphasis, 11px caption. Sheet numbers in JetBrains Mono. Everything else Inter. Max 2 weights per view.

5

8px Spacing Grid

Everything is a multiple of 8: 8, 16, 24, 32, 48. No exceptions. This creates the "calm alignment" that Linear achieves.

6

200ms Transition Max

Ease-out curves only. No bounce, no spring, no playful effects. Professional, precise, confident.

7

Dark Viewer, Light List

Browser is light (familiar). Viewer is dark (professional, reduces eye strain). Clear psychological boundary between browsing and working.

8

One Action Per State

ONE primary action at any moment. Upload when empty. View when browsing. Markup when viewing. Never compete for attention.

05 — Core Principles

Design Principles

Every decision passes through these five filters

01

The Drawing IS the Interface

The plan itself is the primary UI element. Everything revolves around the plan — not sidebars, not tables.

02

Progressive Disclosure

Show the minimum. Reveal complexity on demand. A super sees sheets. A PM sees metadata. Both are served.

03

Spatial > Tabular

Construction people think in space, not spreadsheets. Grid view default. Thumbnails are the navigation.

04

Zero-Friction Transitions

Opening a drawing should feel like zooming into a map — not opening a new app. Seamless spatial continuity.

05

AI as Invisible Infrastructure

AI works silently — auto-classify, auto-link, detect changes. Not a feature you "use" but intelligence you feel.

Tiled Deep Zoom

OpenSeadragon loads only visible tiles at the current zoom level. Smooth zoom from overview to reading 1/16" dimension text.

Floating Markup Toolbar

Figma-style tool palette that lives beside the drawing. One click to annotate. No modals, no context switching.

Sheet Navigation Strip

Horizontal thumbnail strip at bottom. Click to jump between sheets. Swipe left/right on mobile.

Minimap Navigator

Always visible. Shows your viewport position. Click anywhere on the minimap to jump there instantly.

Callout Hyperlinking

AI detects "SEE DETAIL A5.01" text and renders clickable hotspots. One tap navigates to the referenced sheet.

Dark Viewer Background

Dark navy (#1a1a2e) behind white drawings. Reduces eye strain, makes the plan pop. Professional feel.

09 — Execution

Implementation Phases

Four weeks to world-class. Each phase delivers standalone value.

Week 1-2

Foundation

"Make it beautiful"

  • Remove metric cards from top
  • Default to grid view with 240px thumbnails
  • Drawing set grouping by discipline
  • Redesign toolbar — contextual, minimal
  • Slide-over detail panel (480px)
  • Skeleton loading states
  • Keyboard shortcuts
  • Consolidate shared constants
  • CSS modules or Tailwind migration
Week 2-3

The Viewer

"Make it magical"

  • Integrate OpenSeadragon
  • Build DZI tile generation pipeline
  • Add minimap navigator
  • Sheet navigation strip
  • Revision diff slider (juxtapose)
  • Floating Figma-style markup toolbar
  • Improved touch gestures
  • Scale-aware measurement tool
Week 3-4

Intelligence

"Make it smart"

  • AI callout hyperlinking
  • Enhanced fuzzy + metadata search
  • Revision change summary AI
  • Auto-organization of uploads
  • Clash detection pins ON drawing
  • Comment threads on coordinates
Week 4

Polish

"Make it insanely great"

  • Grid-to-viewer zoom transition
  • Progressive image loading
  • Dark mode for viewer
  • Offline tile caching
  • Follow mode (collaboration)
  • Activity feed on drawings
  • Deep link sharing
