# Executive Guide

You own the portfolio. SiteSync gives you one screen — KPIs across every project, ranked by risk — and a search box that crosses every project boundary while respecting RLS.

## Portfolio dashboard

The dashboard is [src/pages/portfolio/PortfolioDashboard.tsx](../../src/pages/portfolio/PortfolioDashboard.tsx). It reads from a materialized view (`project_health_summary`) created in [supabase/migrations/20260502120004_portfolio_health_view.sql](../../supabase/migrations/20260502120004_portfolio_health_view.sql), refreshed every 5 minutes by [supabase/functions/portfolio-summary-refresh/index.ts](../../supabase/functions/portfolio-summary-refresh/index.ts) (cron registered at [supabase/migrations/20260502130000_portfolio_summary_refresh_cron.sql](../../supabase/migrations/20260502130000_portfolio_summary_refresh_cron.sql)).

The pure portfolio library:

- Health rollup (per-project metrics → org KPIs): [src/lib/portfolio/healthRollup.ts](../../src/lib/portfolio/healthRollup.ts)
- Risk ranker (0-100 score, green/yellow/red, factor list): [src/lib/portfolio/riskRanker.ts](../../src/lib/portfolio/riskRanker.ts)
- Index: [src/lib/portfolio/index.ts](../../src/lib/portfolio/index.ts)

Risk weighting per [docs/ENTERPRISE_ADOPTION_PACK.md](../ENTERPRISE_ADOPTION_PACK.md): schedule slip > safety > RFIs > margin > pay app > budget variance.

## What you'll see

- KPI tiles across the org (total active value, projects in critical risk, this-week pay-app dollars in flight)
- A risk-ranked table of projects
- A trend line per KPI

The dashboard is NOT yet route-registered per [STATUS.md](../STATUS.md). Once registered at `/portfolio/dashboard`, it lazy-loads.

## Cross-project search

[src/pages/portfolio/CrossProjectSearch.tsx](../../src/pages/portfolio/CrossProjectSearch.tsx) calls [supabase/functions/cross-project-search/index.ts](../../supabase/functions/cross-project-search/index.ts), which calls the SQL function `search_org()`. The SQL function joins `project_members.user_id = auth.uid()` server-side, so even an executive only sees results from projects they belong to. RLS is enforced at the SQL function level, not the edge function — a compromised edge function cannot expose inaccessible projects.

The search is full-text. Index migration: [supabase/migrations/20260503110000_fts_indexes.sql](../../supabase/migrations/20260503110000_fts_indexes.sql). The org search index view: [supabase/migrations/20260502120005_org_search_index.sql](../../supabase/migrations/20260502120005_org_search_index.sql).

## Owner weekly digest

If you are the executive sponsor on a project, you can opt into the owner weekly digest, generated by [supabase/functions/owner-weekly-digest/index.ts](../../supabase/functions/owner-weekly-digest/index.ts) using [src/lib/documentGen/ownerWeeklyDigest.ts](../../src/lib/documentGen/ownerWeeklyDigest.ts). The digest is content-hashed for tamper evidence (see [docs/PLATINUM_AI_PRODUCTIVITY.md](../PLATINUM_AI_PRODUCTIVITY.md)).

## Forecasting and predictions

The risk engine is [src/lib/riskEngine.ts](../../src/lib/riskEngine.ts); predictions live at [src/lib/predictions.ts](../../src/lib/predictions.ts). Health scoring is [src/lib/healthScoring.ts](../../src/lib/healthScoring.ts) and [src/lib/scheduleHealth.ts](../../src/lib/scheduleHealth.ts).

The "Forecast" tab in the workforce module is currently demo-only — see [src/pages/dashboard/index.tsx](../../src/pages/dashboard/index.tsx) and the recent commit history. Treat its numbers as illustrative until that tab is wired to live data.

## What you should not expect (yet)

- **Cross-portfolio AI insights** — the insight generator at [supabase/functions/generate-insights/index.ts](../../supabase/functions/generate-insights/index.ts) exists but is not surfaced widely; per [HONEST_STATE.md](../../HONEST_STATE.md) the AI is "infrastructure, not UX" today.
- **Bitemporal "as of" reporting** — the codebase doesn't model valid time vs transaction time; the dashboard reflects current state only.
- **Live consolidation across multiple SiteSync orgs** — one portfolio = one organization.
