## How the regression catch actually works

A PR that touches `src/lib/iris/style.ts` (e.g. softening `no-em-dash` to also accept en-dash):

1. `iris-eval` job's `Detect Iris-relevant changes` step matches the modified file path.
2. `Validate corpus is in sync with generator` re-runs `build-iris-corpus.ts` and asserts no diff.
3. `Typecheck Iris eval modules` runs `npm run typecheck` (covers `src/`).
4. `Run live Iris eval` calls staging `iris-call` for all 30 rows. The voice assert calls `lintVoice()` (the same function the change modified). Rule fires fewer than expected â†’ `irisVoiceLintPassRate` drops below 0.95 â†’ `Check eval gate` fails â†’ PR red.

A PR that touches a **prompt** (e.g. weakens the no-em-dash promptBlock):

1. Same path detection.
2. Live eval runs. The model now actually emits em-dashes. The post-process linter still autofixes them (no-em-dash has a suggestedReplacement), so the linter's `passed` stays true. BUT the action assert's `mustNotContain: 'â€”'` checks the final user-visible text â†’ catches the case where the linter's autofix leaves stray markdown em-dashes inside, e.g. inside a code fence the linter doesn't visit.

This is the "two layers of defense" pattern from ADR-005, now graded automatically.

## Risk register

- **Self-host adds ops surface.** Mitigated: Fly.io single-region deploy + managed PG + documented restore drill (matches the `restore-drill.yml` cadence). Cost: ~$5/mo at pilot volume.
- **CI cost.** Mitigated: 30-row dataset + concurrency 5 (~$0.30/run on Sonnet). Workflow is path-filtered so only Iris-touching PRs trigger the live eval.
- **Synthetic corpus has lower fidelity than real PM hand-edits.** Acknowledged. Mitigated: corpus is **additive** â€” when Days 43â€“49 produces real hand-edit pairs, they go into `hand-edits.jsonl` with strict mode (every voice rule must pass). Synthetic rows pruned in a follow-up. The pass-rate floor is conservative (0.85) precisely because synthetic prompts are a coarser signal.
- **Pilot data privacy.** Self-host (ADR-022) keeps trace data on Walker-controlled infra. Traces tagged with `is_soft_pilot` per ADR-006 so retention can be tuned per ADR-008 (24 months for pilot, anonymize after).

## Out of scope (explicit follow-ups)

- **Direct-Anthropic edge function instrumentation** (`draft-daily-log`, `voice-extract`, `agent-orchestrator`). The canonical `iris-call` path is instrumented; the direct paths are server-internal pipelines that deserve their own review pass. Tracked as a follow-up.
- **Browser-side `recordIrisScore` UI** â€” accept/reject buttons emitting a Langfuse score require a new edge function (we cannot ship a secret to the browser; the existing `recordCorrection` writes to `training_corrections` and is the de-facto bridge today).

- **Promptfoo as a CI dependency.** Not added â€” the YAML is dev-only, npx-runnable. The runner (`run-iris-eval.ts`) is the CI surface and is fully type-safe.
- **Replacing `traceLLM` / `ai_cost_tracking`.** Both stay. Langfuse is additive.
