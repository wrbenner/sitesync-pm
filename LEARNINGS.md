## What Works
- Single-file, targeted prompts with exact function/component names get committed successfully
- ARIA fixes, loading states, and mobile responsiveness are reliable Surgeon mode wins
- Commit messages with issue IDs (e.g. "engine: fix ui-design-system-C1-002") enable accurate verification
- Following existing patterns from CODEBASE_PATTERNS.md prevents type errors and style inconsistencies

## What Fails
- Prompts that reference multiple files at once tend to fail or only partially apply
- Prompts that say "refactor" without specifying exact before/after states produce regressions
- Creating new files without adding proper TypeScript types causes build failures
- Installing npm packages without importing them correctly leads to wasted prompts on build fixes
