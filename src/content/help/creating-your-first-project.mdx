# Creating your first project

A project in SiteSync is the central record everything else hangs off — RFIs, drawings, budget, schedule, daily logs, the audit trail. This guide walks you through setup end-to-end.

## Where to click

1. From any page, click the **+** menu in the top bar, then **New Project**. (Alternatively: **The Day → New Project** card in the upper right.)
2. The project creation modal opens with a four-tab flow: Basics, Location, Financials, Team.

## Tab 1 — Basics

Fill in:

- **Project name** — Internal name; show on all reports.
- **Project number** — Optional. Auto-generated as `YYYY-NNN` if blank.
- **Project type** — Commercial, residential, civil, industrial, or institutional. Drives default cost code template and report formatting.
- **Delivery method** — Lump sum, GMP, cost-plus (T&M), design-build, IPD. Drives change order workflow and budget views.
- **Start date** and **Target completion date**.
- **Project description** — Free text. Surfaces in IRIS context.

### Project type vs delivery method

These are independent axes. A *commercial* project (type) can be *lump sum* or *cost-plus* (delivery). The type controls the report template; the delivery method controls how change orders, billing, and budget pages behave. Don't conflate them.

## Tab 2 — Location

- **Address** — Start typing, autocomplete picks up. SiteSync geocodes via Mapbox to set lat/long for weather, sunrise/sunset, and timezone.
- **Site contact** — Person and phone for field deliveries.
- **Timezone** — Auto-detected from address; override if the office runs on a different zone.

### Geocoding gotchas

- Rural sites, new subdivisions, and military bases often fail autocomplete. Click **Manual coordinates** and paste lat/long from Google Maps (right-click any point, click coords to copy).
- An incorrect geocode breaks weather pulls in The Field — daily log weather will read from the wrong city.
- After saving, verify the pin on the project map. Drag to correct if needed.

## Tab 3 — Financials

- **Original contract value** — Used as denominator for change order percentages and burn-rate views.
- **Cost code template** — CSI MasterFormat (16-division or 50-division), copy from existing project, or upload CSV.
- **Retention %** — Default 10%, override per state convention.
- **Tax rate** — Used for budget rollups, not applied to billing line items.

## Tab 4 — Team

Add the project owner (defaults to creator) and one project manager. Full team invites can wait — see [Inviting your team](./inviting-your-team) for the full flow including superintendents and subs.

## What gets auto-created on save

The instant you click **Create Project**, SiteSync provisions:

- **RFI numbering scheme** — `RFI-001` ascending, padded to three digits, per-project counter.
- **Submittal numbering** — `SUB-001` ascending, per-project.
- **Change order numbering** — `CO-001` ascending, per-project.
- **Default cost codes** — From the template you chose.
- **Empty schedule** — Ready for import or manual entry.
- **Document folders** — Contracts, Drawings, Submittals, RFIs, Daily Logs, Photos, Meeting Minutes, Permits.
- **Audit log** — Hash-chained from project creation event. Every subsequent change links cryptographically to the prior entry.
- **IRIS context** — Project name, type, delivery, and description are fed to IRIS for grounded responses.

## After saving

You'll land on **The Day** filtered to the new project. Recommended next steps:

1. Upload the contract drawings to **The Drawings**. They'll OCR overnight and become searchable.
2. Import the schedule from MS Project or P6 (`.xer`, `.xml`, or `.mpp`) via **The Plan → Import**.
3. Invite your team via **The Crew → Invite**.
4. Configure RFI SLAs in **Settings → Project SLAs** if you want non-default ladders. See [RFIs 101](./rfis-101).
