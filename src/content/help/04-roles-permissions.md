# Understanding roles and permissions

SiteSync uses two layers: **organization roles** (workspace-wide) and **project roles** (per-project).

## Organization roles

| Role | Can do |
|---|---|
| Owner | Everything, including deleting the workspace |
| Admin | Manage projects + team, billing, settings |
| Member | Work on assigned projects |

## Project roles

| Role | Can do |
|---|---|
| Owner | Full control of one project |
| Admin | Manage project members, settings, approve RFIs |
| Member | Create + edit RFIs, daily logs, etc. |
| Viewer | Read-only |

## Money + schedule + field actions

Buttons that touch money (budget edits, change orders, payment apps), the schedule, or field operations require admin or owner role on the project.
