# Iris eval corpus

This directory holds the corpus that `evals/iris/promptfooconfig.yaml` runs
against. The pipeline is described in
`docs/audits/IRIS_EVAL_PIPELINE_SPEC_2026-05-08.md`.

## Files

| File | Purpose | Source |
|---|---|---|
| `seed.jsonl` | The 30-row synthetic seed corpus. Regenerated by `scripts/build-iris-corpus.ts`. | `synthetic` |
| `hand-edits.jsonl` | (Created during Days 43–49.) Real PM hand-edits absorbed into the corpus. | `hand_edit` |

`promptfooconfig.yaml` reads both files (`hand-edits.jsonl` is optional).

## Schema

Each row matches `EvalCorpusRow` in `evals/iris/types.ts`. JSONL — one
object per line. Add new rows by appending to the appropriate file; the
`id` field must be unique across both files.

## Growth plan: 30 → 150

The 30 synthetic rows cover the 6 `DraftedActionType` values × 5
confidence buckets. Each row targets one or more voice rules from
`src/lib/iris/style.ts` so a regression in any rule surfaces immediately.

The Days 43–49 hand-edit cycle (per
`docs/audits/IRIS_VOICE_GUIDE_SPEC_2026-05-04.md`) adds up to 150 real
hand-edited rows. Promotion path:

1. PM corrects an Iris draft in production.
2. The diff lands in `iris_voice_diffs`.
3. `scripts/sample-voice-corpus.ts` stratifies + samples.
4. The selected rows are appended to `hand-edits.jsonl` with
   `source: "hand_edit"`, `groundTruthOutput` populated, and
   `expected.voiceRuleIds: []` (strict — every rule must pass on real
   PM-corrected text).

Synthetic rows stay in the corpus until the hand-edit count for their
action_type + confidence band reaches the target floor. They are then
optionally pruned in a separate PR — the eval pipeline does not care
about source provenance, only that every row meets its asserts.

## Adding a synthetic row

Edit `scripts/build-iris-corpus.ts` and re-run `npm run eval:iris:expand`.
The script is idempotent — existing rows with the same `id` are
overwritten in place; new rows are appended; removed rows are deleted.

## Adding a hand-edit row

Append a JSON object to `hand-edits.jsonl`. Validate locally with:

```bash
npm run eval:iris  # parses both files; surfaces shape errors loudly
```
